Estender o backend de empresas para o modelo de 3 responsáveis por setor: Zod schema com 3 campos distintos + CLT; `listarResponsaveis(setor?)` filtrando por setor; `EMPRESA_SELECT` expondo a relação junction; e `criarEmpresa`/`editarEmpresa` gravando Empresa + até 3 linhas `EmpresaResponsavelSetor` numa única transação, com guard server-side DONO-only (D-02) e escrita em lockstep de `responsavelId` legado.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client form → Server Action | FormData (incl. 3 responsável ids + CLT) is untrusted; validated + role-gated server-side |
| Server Action → Postgres | multi-table write (Empresa + up to 3 junction rows) must be atomic |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-10 | Elevation of Privilege | non-DONO submits responsavelFiscal/Dp/ContabilId via direct Server Action call, bypassing UI-disabled selects | mitigate | Server-side `session.user.role === "DONO"` check before accepting any responsável field change; non-DONO changes stripped silently; new test in `empresas.idor.test.ts` asserts the value is not applied (D-02) |
| T-05-11 | Information Disclosure / IDOR | colaborador edits/reads empresa outside their sector scope | mitigate | `editarEmpresa` keeps `findFirst({ where: { id, ...withVisibilityScope(session.user) } })` returning "não encontrado" (never 403) before any write; queries use sector-aware scope from Plan 02 |
| T-05-12 | Tampering | partial write leaves Empresa updated but junction rows inconsistent | mitigate | Empresa write + up-to-3 junction upserts wrapped in a single `db.$transaction`; lockstep `responsavelId` = `responsavelFiscalId` in same tx prevents drift |
| T-05-13 | Information Disclosure | `senhaHash` exposed via `responsaveisPorSetor.usuario` relation in EMPRESA_SELECT | mitigate | Nested select restricted to `{ id, nome }`; no spread of the Usuario row |
| T-05-14 | Input Validation | malformed CLT/responsável payload persisted | mitigate | `empresaSchema.safeParse` (Zod) validates before any DB write; CLT coerced to boolean, Fiscal required, DP/Contábil optional |
| T-05-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages installed this phase; no install task present |
