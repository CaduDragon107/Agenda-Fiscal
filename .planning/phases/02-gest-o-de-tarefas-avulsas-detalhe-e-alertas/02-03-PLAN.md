### A. `src/app/(app)/tarefas/page.tsx` (Server Component)

Sem `"use client"` — este é um Server Component.

Estrutura do componente:
```
export default async function TarefasPage() {
  const session = await auth()
  if (!session?.user) redirect("/login")

  const [tarefas, responsaveis, empresas] = await Promise.all([
    listarTarefas(session.user),
    listarResponsaveis(),        // de @/modules/empresas/queries — lista todos usuários (per D-15)
    listarEmpresas(session.user) // de @/modules/empresas/queries — empresas do escopo do usuário
  ])

  return (
    <div className="flex flex-col gap-6 p-6">
      <div className="flex items-center justify-between">
        <h1 className="text-xl font-semibold">Tarefas</h1>
        <NovaTarefaDialog responsaveis={responsaveis} empresas={empresas} />
      </div>
      <TarefasTable tarefas={tarefas} responsaveis={responsaveis} isDono={session.user.role === "DONO"} />
    </div>
  )
}
```

async function onSubmit(data) {
  setIsPending(true)
  // Criar FormData a partir dos dados validados pelo RHF
  const formData = new FormData()

  formData.set("empresaId", data.empresaId)

  toast.success("Tarefa criada com sucesso.")

}

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → NovaTarefaDialog → criarTarefa action | FormData gerada client-side não confiável; tarefaSchema valida no server |
| Browser → TarefasTable checkbox → concluirTarefa | id da tarefa enviado do cliente; anti-IDOR no server impede acesso cruzado |
| AppLayout server → contarAlertasTarefas → AppSidebar | query escopada server-side; contador não vaza dados de outros usuários |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-IDOR-UI | Elevation of Privilege | Botão Trash2 na TarefasTable | mitigate | UI oculta o botão para COLABORADOR em tarefas alheias (per UI-SPEC RBAC); server-side anti-IDOR em `excluirTarefa` é a barreira real — a UI é Defense in Depth |
| T-02-CLIENT-FILTER | Elevation of Privilege | Filtro client-side na TarefasTable | accept | `dadosFiltrados` filtra apenas o dataset retornado por `listarTarefas(session.user)`, que já está escopado server-side. Impossível ver tarefas fora do escopo via filtro client-side |
