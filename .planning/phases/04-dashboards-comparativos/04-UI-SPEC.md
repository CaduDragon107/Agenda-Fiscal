## Spacing Scale

Declared values (must be multiples of 4) — matches existing project usage (`tarefas-table.tsx`, `app-sidebar.tsx`):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gaps (`gap-1`), badge internal padding |
| sm | 8px | Compact toolbar element spacing (`gap-2`), chart legend gaps |
| md | 16px | Default card padding, gap between stacked dashboard sections within a card (`gap-4`) |
| lg | 24px | Section padding inside each dashboard `Card` |
| xl | 32px | Gap between the 3 stacked dashboard sections on `/dashboards` page |
| 2xl | 48px | Not used this phase |
| 3xl | 64px | Not used this phase |

Exceptions: Chart containers require an explicit minimum height outside the 8-point scale rhythm — `min-h-[260px]` for bar/line/area charts (Pitfall 4 in RESEARCH.md: omitting this renders a 0px chart). This is a sizing constraint, not a spacing token, and is declared once here so the executor does not improvise a different value per chart.

## Typography

Reuses the project's existing type scale (no new sizes introduced — confirmed by scanning `tarefas-table.tsx`, `app-sidebar.tsx`: only `text-xs`, `text-sm`, `text-xl font-semibold` appear project-wide for the roles below; `font-medium`/500 is NOT used by this phase, see note below).

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body / table cells | 14px (`text-sm`) | 400 (regular) | 1.5 |
| Label / metadata / muted context (volume counts, "X empresas") | 12px (`text-xs`) | 400 (regular) | 1.5 |
| Card title / chart heading | 16px (`text-base font-semibold`) | 600 (semibold) | 1.2 |
| Page heading ("Dashboards") / KPI headline figure | 20px (`text-xl font-semibold`) | 600 (semibold) — matches empty-state heading pattern (`text-xl font-semibold`) in `tarefas-table.tsx` | 1.2 |

Declared sizes: 12, 14, 16, 20px — exactly 4 sizes total this phase, within the 3-4 ceiling. The KPI headline figure (e.g., "87% no prazo") reuses the 20px/600 page-heading role rather than introducing a separate "Display" size — there is no 5th size this phase. Declared weights: 400 regular, 600 semibold — exactly 2 weights total this phase. The Card title / chart heading role uses 600 (`font-semibold`), not the project's pre-existing `font-medium`/500 utility; `font-medium` remains in use elsewhere in the codebase (e.g., table headers/links) but is explicitly NOT part of this phase's active type roles, so it does not count toward this phase's weight budget.

## Color

Reuses the project's existing OKLCH greyscale token system in `src/app/globals.css` — no new colors introduced. The project currently has zero hue (all `oklch(L 0 0)` — pure greyscale) for primary/secondary/chart tokens; `--destructive` is the only chromatic token (red, `oklch(0.577 0.245 27.325)`).

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--background)` (white / near-black in dark mode) | Page background, card interiors |
| Secondary (30%) | `var(--card)`, `var(--secondary)`, `var(--sidebar)` | Dashboard `Card` surfaces, sidebar nav, table header rows |
| Accent (10%) | `var(--chart-1)` through `var(--chart-5)` (greyscale ramp, oklch 0.87 → 0.269) | **Reserved exclusively for:** (1) the primary bar/line series in each of the 3 charts (DASH-01 bar, DASH-02 line/area, DASH-03 ranking bar), (2) the headline KPI percentage figure text color when it needs to stand out from body text. Do NOT use chart colors for any other UI chrome (buttons, borders, nav). |
| Destructive | `var(--destructive)` | Reserved for "atrasado"/overdue indicators only — reuses the exact same token and `Badge` `variant="destructive"` pattern already established in `calcularAlertaPrazo`/`PrazoCell` (Phase 2). Used in: overdue badge in DASH-03 ranking table, red downward-trend indicator if a colaborador's % no prazo drops month-over-month. |

**No new semantic color is introduced.** Good performance (high % no prazo) uses the existing neutral/default badge styling already used for "concluída" elsewhere — this project deliberately does not have a "success green," consistent with the existing greyscale palette; do not invent one for this phase. Trend direction (up/down) is communicated via icon (`TrendingUp`/`TrendingDown` from lucide) and text, not via a new green/red pairing beyond the existing destructive-for-bad convention.

**Dark mode:** All `--chart-1..5` and semantic tokens already have `.dark` overrides in `globals.css` — charts must reference `var(--chart-N)` directly (not `hsl(var(--chart-N))`), per RESEARCH.md's State of the Art note, since this project's tokens are raw `oklch()` values.

## Layout & Component Inventory

Reuses established project patterns — no new layout primitives introduced.

| Need | Component | Source |
|------|-----------|--------|
| Page shell | Existing `(app)` layout + `AppSidebar` | Already built (Phase 1); enable the currently-`disabled` "Dashboards" `SidebarMenuButton` in `app-sidebar.tsx`, gated by `isDono` (hide entirely for COLABORADOR, matching how other role-specific nav items would be hidden — defense in depth only, real gate is server-side) |
| Section containers | `Card` / `CardHeader` / `CardTitle` / `CardContent` (shadcn) | `src/components/ui/card.tsx` (already installed) |
| Charts | New `ChartContainer` + Recharts `BarChart`/`LineChart`/`AreaChart` (per RESEARCH.md Patterns 4/5/7) | `npx shadcn add chart` (to install this phase) |
| Ranking table (DASH-03, full list) | `Table`/`TableHeader`/`TableBody`/`TableRow`/`TableCell` + TanStack Table | Reuse exact pattern from `tarefas-table.tsx`/`empresas-table.tsx` — sortable column, no pagination required at ~100-110 rows (RESEARCH.md Pattern 7 recommends top-N chart + full table) |
| Overdue/badge indicators | `Badge` with `variant="destructive"` | `src/components/ui/badge.tsx` (already installed) — reuse `calcularAlertaPrazo`-style badge convention |
| Period selector (DASH-02) | `Select`/`SelectTrigger`/`SelectContent`/`SelectItem` | `src/components/ui/select.tsx` (already installed, same pattern as the status filter in `tarefas-table.tsx`) |
| Page-level loading state | `Skeleton` (shadcn) matching the `loading.tsx` pattern from `tarefas/[id]/loading.tsx` | `src/components/ui/skeleton.tsx` (already installed) |

**Layout structure (per RESEARCH.md Assumption A3 — Claude's Discretion, confirmed here):** Single `/dashboards` page with 3 stacked `Card` sections in vertical order (Desempenho por colaborador → Evolução mensal → Empresas com mais atrasos), not tabs and not separate routes. Each section is full-width on desktop; charts use `min-h-[260px]` per the spacing exception above. This avoids tab-state complexity for a DONO-only, low-traffic page and keeps all three comparative views scannable in one scroll, consistent with the "dono sempre sabe em tempo real o status de tudo" core value (PROJECT.md) — a single glance-able page serves that value better than navigation-gated tabs.
