<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser (button click) → Server Action | Untrusted client invokes gerarTarefasDoMesAction; a COLABORADOR can call it directly even with the button hidden |
| Server Action → executarGeracaoMensal → DB | Authorized DONO-only write path |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-3-01 | Elevation of Privilege | gerarTarefasDoMesAction called directly by a COLABORADOR despite hidden button | mitigate | Server-side `if (session.user.role !== "DONO") return { ok: false, error: "não autorizado" }` as the FIRST check after `auth()`, BEFORE any DB access — enforced in the action, never only by hiding the button client-side (V4 Access Control); covered by the RBAC test asserting createMany is not called |
| T-3-05 | Spoofing / unauthenticated access | gerarTarefasDoMesAction called without a session | mitigate | `auth()` guard returns `{ ok: false, error: "Não autenticado" }` before role check, same as criarTarefa (V2) |
| T-3-06 | Tampering | non-canonical competência string defeating idempotency | mitigate | If the action accepts any competência input it is validated with `competenciaSchema` (`/^\d{4}-(0[1-9]|1[0-2])$/`); default path uses `competenciaAtual()` (date-fns format) which is always canonical (V5, Pitfall 4) |
</threat_model>
