## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → upload de arquivo | .xlsx não confiável; pode ser arquivo malicioso disfarçado |
| dados staged → persistência | só persistir após revisão humana e confirmação explícita |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-UPLOAD | Tampering | parseUploadAction (XLSX.read) | mitigate | validar extensão/MIME .xlsx + try/catch em XLSX.read; erro genérico "Arquivo inválido..." sem expor detalhe do parser (Task 2) |
| T-01-IMPORT-AUTHZ | Spoofing | actions de importação | mitigate | auth() guard no topo de parseUploadAction e confirmarImportacao |
| T-01-IMPORT-INPUT | Tampering | confirmarImportacao | mitigate | empresaSchema valida cada linha (CNPJ módulo 11 + regime enum + responsavelId obrigatório) antes do create; revisão humana bloqueia confirmação com dados ausentes |
| T-01-IMPORT-CSRF | Tampering | Server Actions de importação | mitigate | proteção CSRF nativa do Next Server Actions; não desabilitar |
</threat_model>
