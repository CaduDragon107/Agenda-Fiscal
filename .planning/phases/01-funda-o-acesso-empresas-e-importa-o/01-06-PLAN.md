<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| internet pública → app | qualquer um na internet pode atingir a URL; auth é a única barreira |
| Railway env vars → app | AUTH_SECRET e DATABASE_URL como segredos de produção |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-PROD-SECRET | Information Disclosure | env vars de produção | mitigate | AUTH_SECRET/DATABASE_URL só em Railway Variables; nunca no repo (apenas .env.example sem valores) |
| T-01-PROD-TLS | Information Disclosure | tráfego público | mitigate | Railway serve HTTPS por padrão no domínio *.up.railway.app; cookie de sessão secure em produção |
| T-01-PROD-AUTHHOST | Spoofing | Auth.js atrás de proxy | mitigate | AUTH_TRUST_HOST=true configurado para o proxy do Railway validar host corretamente |
| T-01-PROD-EXPOSURE | Information Disclosure / Spoofing | app público sem auth | mitigate | middleware.ts (Plano 02) protege todas as rotas exceto /login; sem login não há acesso a dados |
</threat_model>
