# Phase 1 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

Phase 1 plan/execution must include project bootstrap (`create-next-app` + `shadcn init`) as a prerequisite task. ## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, inline tag spacing |
| sm | 8px | Compact element spacing (form field internal padding, table cell vertical padding) |
| md | 16px | Default element spacing (gaps between form fields, card padding, list item padding) |
| lg | 24px | Section padding (card group spacing, page section gaps) |
| xl | 32px | Layout gaps (between major page regions, sidebar/main gutter) |
| 2xl | 48px | Major section breaks (between page header and content body) |
| 3xl | 64px | Page-level spacing (top padding on auth/login page, empty-state vertical centering) |

Exceptions:
- **44px minimum touch target** for icon-only buttons (table row actions: edit/delete icons, sidebar nav icons on mobile) — accessibility minimum, applies even though it's not a multiple of 4.
- **Login page card max-width: 400px**, centered — not a spacing token, but a layout constraint for the auth screen.
- **Sidebar width: 240px** (desktop, collapsible to icon-only 64px) — layout constant, not part of the 8pt content spacing scale.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | Regular (400) | 1.5 |
| Label | 14px | Semibold (600) | 1.5 |
| Heading | 20px | Semibold (600) | 1.2 |
| Display | 28px | Semibold (600) | 1.2 |

**Usage guide:**
- **Body (14px/400)**: table cell content, form input text, descriptive paragraphs, helper text.
- **Label (14px/600)**: form field labels, table column headers, badge text, nav item labels.
- **Heading (20px/600)**: section titles ("Empresas", "Importar planilha", "Editar empresa"), card titles, dialog titles.
- **Display (28px/600)**: page-level titles ("Empresas-cliente", login page title "Agenda Fiscal").

**FLAG:** shadcn's default base font size is 16px (Tailwind `text-base`). This spec deliberately sets **Body to 14px** (Tailwind `text-sm`) as the dominant size because the empresas table will display 100+ dense rows with many columns (nome, CNPJ, regime, responsável, contatos) — 14px keeps rows compact without horizontal scroll on laptop screens. This is a common, accepted density choice for admin/data tables (matches shadcn's own `data-table` examples, which default table cells to `text-sm`). Only 3 sizes declared (14/20/28) plus the 14px label at a different weight — stays within the "3-4 sizes, 2 weights" budget (sizes: 14, 20, 28 = 3; weights: 400, 600 = 2).

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FFFFFF` (light) / shadcn `background` token — `oklch(1 0 0)` | Page background, dialog/card backgrounds, table row background (odd rows) |
| Secondary (30%) | `#F4F4F5` (Tailwind zinc-100) / shadcn `muted` + `card` tokens | Sidebar background, table header row, card backgrounds, alternating table row stripe (even rows), input backgrounds |
| Accent (10%) | `#18181B` (Tailwind zinc-900, shadcn default `primary` in Neutral preset) | **Reserved for:** primary CTA buttons only — "Entrar" (login submit), "Salvar empresa" (create/edit form submit), "Confirmar importação" (import confirm step), "Nova empresa" (page-level add button), active sidebar nav item indicator, focus ring on inputs |
| Destructive | `#DC2626` (Tailwind red-600, shadcn default `destructive` token) | **Reserved for:** "Excluir empresa" button/icon, CNPJ-inválido row badges in import preview, "atrasado"/error banner backgrounds, error toast/alert text |

Accent reserved for: **primary CTA buttons** ("Entrar", "Salvar empresa", "Nova empresa", "Confirmar importação", "Avançar" in the import wizard), **active navigation state** in the sidebar, and **focus rings** on form inputs. Never used for body text, decorative elements, or secondary/tertiary buttons (those use `variant="outline"` or `variant="ghost"` in shadcn, which stay neutral/secondary).

**FLAG:** Standard shadcn Neutral-preset palette (zinc-based grayscale + near-black accent) chosen as the default — neutral, professional, low-risk for an accounting back-office tool. No brand colors were specified upstream (CONTEXT.md does not exist for this phase). If the office has an existing brand color (logo, letterhead), it can be swapped into the `--primary` CSS variable later (single-point change in `globals.css`, shadcn theming is variable-driven) without touching component code. Not blocking.

**Semantic status colors (used for ALRT-01-adjacent badges, e.g. regime tributário tags and import report rows — not full alert system, which is Phase 2):**
| Status | Color | Usage |
|--------|-------|-------|
| Success / valid | Tailwind `green-600` (`#16A34A`) | "CNPJ válido" indicator, "Importação concluída" success state |
| Warning / needs review | Tailwind `amber-500` (`#F59E0B`) | "Regime tributário ausente" badge in import preview, "Revisar antes de confirmar" banner |
| Info / Lucro Real tag | Tailwind `blue-600` (`#2563EB`) | Regime tributário badge: "Lucro Real" |
| Info / Simples Nacional tag | Tailwind `purple-600` (`#9333EA`) | Regime tributário badge: "Simples Nacional" |

These four are used sparingly as **badge fill colors only** (regime tags, import-row status) — they do not compete with the 10% accent for CTAs.

## Phase 1 Screen Inventory

This section extends the template to give the planner/executor a concrete component-and-state inventory for this phase's UI surface, derived from the success criteria (AUTH-01, AUTH-02, EMPR-01, EMPR-02, INFRA-01).

### 1. Login screen (`/login`)
- **Layout:** Centered card, max-width 400px, vertical centering with 64px (3xl) top padding on larger viewports.
- **Components:** Card, Form (email + password fields via `react-hook-form` + `zod`), Input, Label, Button (primary, full-width, "Entrar").
- **States:**
  - Default: empty form, focus on email field on load.
  - Validation error (client-side, Zod): inline red text under field ("Email inválido", "Senha obrigatória").
  - Submit error (server): inline error message above the submit button — copy: "Email ou senha incorretos." (do not reveal whether email exists, for security).
  - Loading: button shows spinner + disabled state during submit.
- **Persistence:** Session persists across browser restarts (Auth.js JWT session, long-lived cookie) — no "remember me" checkbox needed (always persistent per AUTH-01).

### 2. Authenticated shell / layout
- **Layout:** Sidebar (240px, collapsible to 64px icon rail on smaller viewports) + main content area. Sidebar background uses Secondary (30%) color.
- **Sidebar contents:**
  - Logo/app name ("Agenda Fiscal") at top.
  - Nav items: "Empresas" (Phase 1 scope). Future phases add "Tarefas", "Dashboards" — nav structure should anticipate these as disabled/hidden items, not require redesign.
  - Role-aware: **dono** sees an additional nav indicator or label distinguishing "visão geral" — but no separate nav items required in Phase 1 beyond what's needed for empresas (full list for dono, scoped list for colaborador — same screen, different data via backend filter).
  - User menu at bottom of sidebar: user name/email, "Sair" (logout) action via DropdownMenu.
- **Components:** shadcn `sidebar` block, DropdownMenu, Avatar (initials fallback, no photo upload in v1).
- **Responsive:** On mobile/narrow viewports, sidebar collapses to a top bar with a hamburger trigger (shadcn sidebar default behavior).

### 3. Empresas list (`/empresas`)
- **Layout:** Page header (Display 28px title "Empresas" + "Nova empresa" primary button, right-aligned) + filter/search bar + TanStack Table.
- **Components:** Table (TanStack Table + shadcn `table` primitives), Input (search by nome/CNPJ), Select (filter by regime tributário: "Todos" / "Lucro Real" / "Simples Nacional"), Select (filter by responsável — only visible/meaningful for dono; colaborador's view is pre-scoped so this filter may be hidden or default to themselves), Badge (regime tributário tag — blue for Lucro Real, purple for Simples Nacional, per Color section), Pagination controls (TanStack Table pagination, page size default 20 rows).
- **Columns:** Nome, CNPJ (formatted with mask `00.000.000/0000-00`), Regime tributário (badge), Responsável, Ações (edit icon button + delete icon button, 44px touch targets).
- **States:**
  - Default: paginated table, 20 rows/page, sortable by Nome (default ascending).
  - Loading: Skeleton rows (shadcn `skeleton`) while data fetches.
  - Empty (no empresas at all): Empty state per Copywriting Contract — heading "Nenhuma empresa cadastrada", body with two CTAs ("Importar planilha" secondary, "Nova empresa" primary).
  - Empty (filtered, no matches): "Nenhuma empresa encontrada" / "Tente ajustar os filtros ou o termo de busca."
  - Row action — delete: opens AlertDialog per Copywriting Contract destructive pattern.
- **Visibility (AUTH-02, backend-enforced):** Colaborador sees only empresas in their carteira; dono sees all. The table component itself is identical for both roles — scoping happens in the data-fetching layer, not via UI conditionals. (Backend enforcement is an architecture concern, not a UI concern, but the UI must NOT assume client-side filtering is sufficient — always render exactly what the backend returns.)

### 4. Empresa create/edit form (`/empresas/novo`, `/empresas/[id]/editar`)
- **Layout:** Card-based form, max-width ~640px, single column on mobile, can use 2-column grid (16px/md gap) for short fields (CNPJ + regime side-by-side) on desktop.
- **Components:** Form (react-hook-form + Zod), Input (nome, CNPJ with mask, responsável, contatos), Select (regime tributário: "Lucro Real" | "Simples Nacional" — required field), Textarea (particularidades — free text), Button (primary "Salvar empresa", secondary "Cancelar" which navigates back to `/empresas`).
- **Fields (per EMPR-01):**
  - Nome (text, required)
  - CNPJ (text with mask `00.000.000/0000-00`, required, validated via módulo 11 algorithm — inline error "CNPJ inválido" if check digits fail)
  - Regime tributário (select, required: "Lucro Real" / "Simples Nacional")
  - Responsável (select or text — populated from the 4 colaboradores + dono; if select, options come from `usuarios` table)
  - Contatos (text or textarea — phone/email, free-form per planilha source data)
  - Particularidades (textarea, optional, free-form notes)
- **States:**
  - Create mode: all fields empty, title "Nova empresa".
  - Edit mode: fields pre-populated, title "Editar empresa: {nome}".
  - Validation errors: inline per-field (Zod schema shared client/server).
  - Save success: toast (sonner) "Empresa salva com sucesso." + redirect to `/empresas`.
  - Save error: toast "Não foi possível salvar. Verifique os dados e tente novamente."

### 5. Excel import flow (`/empresas/importar`) — addresses EMPR-02 + Pitfall 7
A 3-step wizard (Tabs or step indicator component — shadcn `tabs` styled as a stepper, OR a simple numbered header "Etapa 1 de 3").

**Step 1 — Upload:**
- Components: Card with file input (drag-and-drop zone + "Selecionar arquivo" button), accepts `.xlsx` only.
- States: idle (empty dropzone), file selected (filename + size shown, "Avançar para revisão" button enabled), error ("Arquivo inválido. Envie um arquivo .xlsx válido.")

**Step 2 — Staging/Preview & human review (the critical step per Pitfall 7):**
- Components: Summary card (import report counts, per Copywriting Contract: "{X} prontas, {Y} CNPJ inválido, {Z} sem regime, {W} duplicadas") with clickable filter chips; editable TanStack Table showing every staged row.
- **Table columns:** Nome, CNPJ (editable, inline validation badge if invalid), Regime tributário (editable Select — **defaults to empty/unset**, since the source planilha covers PIS/COFINS = Lucro Real only and likely lacks this column; per PROJECT.md research flag, Simples Nacional companies are likely NOT in this spreadsheet at all, so most/all imported rows may need "Lucro Real" confirmed manually or left for the dono to verify), Responsável (editable Select), Contatos, Particularidades, Status badge (one of: "Pronta" / green, "CNPJ inválido" / amber, "Sem regime" / amber, "Duplicada" / amber), row-level checkbox to include/exclude from final import.
- **Row editing:** Inline cell editing (click cell → becomes input/select) — no separate modal per row, since 100+ rows need fast review.
- **Validation (live, client-side):** CNPJ módulo 11 check on edit; duplicate detection by CNPJ highlights all duplicate rows with the same indicator.
- **Required-before-confirm rule:** "Confirmar importação" button is disabled (or shows a warning) while any included row has "Sem regime" status — the helper text "Obrigatório — selecione o regime tributário desta empresa para continuar" appears under that row's regime select.
- **Bulk actions:** "Marcar todas como Lucro Real" quick action (since the planilha is PIS/COFINS-only = likely all Lucro Real) — a secondary button above the table, explicitly labeled so it's an opt-in bulk edit, not a silent default.

**Step 3 — Confirm & result:**
- Components: Final summary card (re-show counts of what will actually be persisted, after exclusions), primary Button "Confirmar importação", secondary Button "Voltar à revisão".
- On confirm: loading state (button spinner + disabled), then success state — Alert (success variant) "Importação concluída: {N} empresas adicionadas." with a link/button to `/empresas`.
- On error (e.g. backend failure mid-import): Alert (destructive variant) with generic error copy + "Tentar novamente" button — staged data is preserved (not lost) so the user doesn't have to re-upload.
- **Discard path:** "Descartar importação" (secondary/ghost button, available from Step 2 or 3) → AlertDialog per Copywriting Contract destructive pattern ("Descartar importação?" / "Os dados revisados serão perdidos...").
