<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → Server Action (CRUD) | empresaId e payload não confiáveis; validados e escopados server-side |
| colaborador → empresa de outro colaborador | tentativa de IDOR via manipulação de id |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-IDOR-MUT | Elevation of Privilege / Information Disclosure | editarEmpresa / excluirEmpresa | mitigate | findFirst({ id, ...withVisibilityScope(session.user) }) antes de toda mutação; retorna "não encontrado" (não 403) se null; coberto por tests/empresas.idor.test.ts |
| T-01-IDOR-READ | Information Disclosure | listarEmpresas na página | mitigate | escopo aplicado na query (Plano 03); página renderiza só o que o backend retorna, sem filtro client-side |
| T-01-CSRF | Tampering | Server Actions CRUD | mitigate | Server Actions do Next têm proteção CSRF nativa (verificação de origem); não desabilitar |
| T-01-INPUT-CRUD | Tampering | criarEmpresa / editarEmpresa | mitigate | empresaSchema.parse server-side (CNPJ módulo 11 + regime enum) antes de persistir |
</threat_model>
