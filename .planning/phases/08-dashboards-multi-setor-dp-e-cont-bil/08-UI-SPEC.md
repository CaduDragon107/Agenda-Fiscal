# Phase 8 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

> **Replication phase notice:** This phase invents no new visual language. It extends the existing `/dashboards` page (Phase 4, Fiscal) with two additional sector tabs (DP, Contábil), reusing the same Card layout, the same 3 chart/table components (parameterized by dataset), and the same traffic-light chart palette. Every value below is either inherited verbatim from the Fiscal precedent or a direct, minimal extension required by D-01/D-04 (tabs, per-sector empty-state copy).

## Spacing Scale

Declared values (must be multiples of 4) — inherited from existing `/dashboards` page, no changes:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps inside sort headers (`gap-1`), badge padding |
| sm | 8px | Compact element spacing (button icon gaps) |
| md | 16px | Card internal padding, table cell padding |
| lg | 24px | Spacing between page header and first Card (`gap-8` container uses 32px; intra-card spacing uses 16-24px) |
| xl | 32px | Vertical gap between stacked Cards (`flex flex-col gap-8` = 32px) |
| 2xl | 48px | Not used on this page |
| 3xl | 64px | Empty-state vertical padding (`py-16` = 64px top/bottom) |

Exceptions: none. Tab bar sits between the page header and the first Card, using the same `gap-8`/32px rhythm as the existing page — no new spacing token introduced.

## Typography

Inherited from existing `/dashboards` page — no new sizes or weights introduced.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|

Only 2 custom weights are used project-wide: 400 (regular) and 600 (semibold). The 500 (medium) weight on `TabsTrigger`/`TableHead` is a shadcn component default, not an additional custom weight choice — do not introduce any other weight.

## Color

Inherited from existing theme (`globals.css` `:root` tokens) — no new colors introduced.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--background` (oklch white / near-white) | Page background |
| Secondary (30%) | `--card`, `--sidebar` | Card surfaces, sidebar, table header row |
| Accent (10%) | `--primary` (neutral-dark, shadcn `neutral` theme) | Active `TabsTrigger` indicator, sort-button hover state, primary buttons (none on this read-only page) |
| Destructive | `--destructive` | `Badge variant="destructive"` on ranking rows ≥ 30% atraso (`LIMIAR_ALTO_ATRASO`) |

Accent reserved for: the active tab's underline/background indicator (`TabsTrigger[data-state=active]`), and table column-sort button hover/focus states. Never used for body text or decorative elements.

**Chart palette (locked, quick-task 260622-r6n — traffic-light scheme, do not alter):**

| Token | Hex (oklch) | Semantic role | Used by |
|-------|-------------|----------------|---------|
| `--chart-1` | `oklch(0.55 0.18 255)` (blue) | "Criadas" / single-series % no prazo / % de atraso | Desempenho chart, Ranking chart |
| `--chart-2` | `oklch(0.62 0.17 150)` (green) | "Concluídas" | Evolução mensal chart |
| `--chart-3` | `oklch(0.80 0.16 95)` (yellow) | "Pendentes (sem motivo)" | Evolução mensal chart |
| `--chart-4` | `oklch(0.68 0.18 55)` (orange) | "Pendentes (com motivo)" | Evolução mensal chart |
| `--chart-5` | `oklch(0.58 0.22 27)` (red) | "Vencidas" | Evolução mensal chart |

This exact palette and category-to-color mapping applies unchanged to DP and Contábil evolution charts — same 5 categories (criadas/concluídas/pendentes-sem-motivo/pendentes-com-motivo/vencidas), same colors, same order.

## Interaction Contract — Tab Navigation (D-01)

| Aspect | Specification |
|--------|----------------|
| Component | shadcn `Tabs` (`src/components/ui/tabs.tsx`, already installed) — `TabsList` + `TabsTrigger` (×3) + `TabsContent` (×3) |
| Placement | Directly below the page header (title + subtitle), above the 3 Cards — same vertical position the Cards currently start at |
| Tabs | "Fiscal", "DP", "Contábil" — in this fixed order (Fiscal first, as the established/proven sector) |
| Default tab | "Fiscal" |
| Tab state persistence | None required — no URL search param or localStorage persistence needed for v1 of this phase; switching tabs is a pure client-side state change. (`searchParams.meses` for the evolução-mensal month-range filter continues to apply per-tab/per-dataset, unaffected by tab selection.) |
| Content per tab | Each `TabsContent` renders the same 3-Card layout (Desempenho / Evolução / Ranking) already used by Fiscal, fed by sector-scoped data from the parameterized `src/modules/dashboards/queries.ts` |
| Active tab indicator | shadcn default — `--primary` colored underline/background per `radix-nova` style, no custom styling |
| Empty dataset per tab | Each Card independently shows its own `EmptyState` (sector-aware copy) if that specific dataset (desempenho/evolução/ranking) is empty — tabs do NOT show a single empty state for the whole sector; each of the 3 cards checks its own array length, exactly as Fiscal does today |
