### Form layout placement (3 responsĂ¡vel selectors, D-02)

Exact placement in `empresa-form.tsx`, replacing the current single "ResponsĂ¡vel" `FormField`:

1. Keep `nome` field as-is (full width, top).
2. Keep `cnpj` + `regimeTributario` 2-column grid as-is.
3. **NEW:** Replace the single "ResponsĂ¡vel" `FormField` with a 3-column grid (`grid grid-cols-1 gap-4 md:grid-cols-3`) containing, in this fixed left-to-right order: **ResponsĂ¡vel Fiscal**, **ResponsĂ¡vel DP**, **ResponsĂ¡vel ContĂ¡bil** â€” Fiscal first because it is the only required field and matches the existing data model's primacy; DP/ContĂ¡bil order follows the same order used throughout CONTEXT.md and ROADMAP.md ("DP/ContĂ¡bil").
4. 5. When `session.user.role !== "DONO"`, all 3 selects render with `disabled` (client-side UX signal only â€” the real enforcement per Pitfall in `05-RESEARCH.md` is server-side in the Server Action, stripping non-DONO field changes silently).
6. Keep `contatos` and `particularidades` fields as-is, below the 3-column grid.
7. **NEW:** Insert the CLT `Checkbox` + helper text as its own `FormItem` immediately after `particularidades`, before the submit button row â€” full width, not part of any grid (matches the existing single-column rhythm of `contatos`/`particularidades`).
