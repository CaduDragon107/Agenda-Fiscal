### Form layout placement (3 responsável selectors, D-02)

Exact placement in `empresa-form.tsx`, replacing the current single "Responsável" `FormField`:

1. Keep `nome` field as-is (full width, top).
2. Keep `cnpj` + `regimeTributario` 2-column grid as-is.
3. **NEW:** Replace the single "Responsável" `FormField` with a 3-column grid (`grid grid-cols-1 gap-4 md:grid-cols-3`) containing, in this fixed left-to-right order: **Responsável Fiscal**, **Responsável DP**, **Responsável Contábil** — Fiscal first because it is the only required field and matches the existing data model's primacy; DP/Contábil order follows the same order used throughout CONTEXT.md and ROADMAP.md ("DP/Contábil").
4. Each of the 3 selects uses placeholder text "Selecione o responsável" (Fiscal — required, matches existing) / "Sem responsável" (DP, Contábil — optional, selectable empty value via a `SelectItem value="">Sem responsável</SelectItem>` first option, mirroring how `StepReview.tsx` represents an unset value as a real selectable `SelectItem` rather than leaving the trigger blank).
5. When `session.user.role !== "DONO"`, all 3 selects render with `disabled` (client-side UX signal only — the real enforcement per Pitfall in `05-RESEARCH.md` is server-side in the Server Action, stripping non-DONO field changes silently).
6. Keep `contatos` and `particularidades` fields as-is, below the 3-column grid.
7. **NEW:** Insert the CLT `Checkbox` + helper text as its own `FormItem` immediately after `particularidades`, before the submit button row — full width, not part of any grid (matches the existing single-column rhythm of `contatos`/`particularidades`).

### Table column placement (D-10, sector-filtered columns)

In `empresas-table.tsx`, the `responsavel` column (currently a single "Responsável" column) is replaced by sector-aware logic:

- **DONO:** renders 3 columns in this fixed order: "Responsável Fiscal", "Responsável DP", "Responsável Contábil" — each cell shows the name or, if null, the "Sem responsável" amber badge (D-03).
- **Colaborador (any setor):** renders exactly 1 column, labeled "Responsável {Setor}" (e.g. "Responsável DP"), showing only that colaborador's own sector's responsável for each visible row — never the other 2 sectors' data, even as a hidden/disabled column (this is a data-exposure boundary, not just a visual one).
- Column order relative to existing columns: Nome, CNPJ, Regime, [Responsável column(s) as above], Ações — i.e. responsável column(s) occupy the same position the single "Responsável" column occupies today, just multiplied for DONO.
