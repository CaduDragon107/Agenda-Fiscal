    Estrutura da planilha `data/EMPRESAS RESPONSÁVEL.xlsx` (já inspecionada — não ler o xlsx novamente):
    - Sheet "Planilha1", linha 0 = título, linha 1 = cabeçalho, linhas 2–198 = dados válidos (CNPJ não vazio), linhas 199–205 = rodapé de totais (ignorar — filtrar por CNPJ vazio)
    - Colunas (0-based): 0=Código, 1=Empresa, 2=CNPJ, 3=Regime Tributário, 4=Responsável

    Leitura da planilha: usar `XLSX.read(buffer, { type: "buffer" })` + `XLSX.utils.sheet_to_json(sheet, { header: 1, raw: false, defval: "" })`. Iterar a partir do índice 2; incluir linha apenas se `String(linha[2]).trim()` for não vazio (filtra automaticamente o rodapé).

    Mapeamento de regime (função `mapearRegime(rotulo)`):
    - "Lucro Real" → `LUCRO_REAL`
    - "Lucro Presumido" → `LUCRO_PRESUMIDO`
    - "Simples Nacional" → `SIMPLES_NACIONAL`
    - "MEI" → `SIMPLES_NACIONAL` (com `particularidades = "MEI"`)
    - qualquer outro → null (reportar como linha inválida, não criar)

    Resolução de responsável: carregar `db.usuario.findMany({ select: { id, nome } })` e construir um Map `nomeParaId` com `nome.trim().toLowerCase() → id`. Buscar `linha[4].trim().toLowerCase()`. Se não resolver → acumular em `responsaveisNaoResolvidos[]`.

    Verificação de idempotência: carregar `db.empresa.findMany({ select: { cnpj: true } })` e montar `Set<string> cnpjsExistentes` com os valores brutos do banco (já persistidos no formato "00.000.000/0000-00"). Para cada linha válida, comparar `linha[2].trim()` com o Set. Match → pular (acumular em `puladas[]`).

    Dry-run report (imprimir antes de sair quando !APPLY):
    - Total linhas válidas (CNPJ não vazio)
    - A criar (novas), a pular (CNPJ duplicado)
    - Linhas MEI (contagem + lista de CNPJs/nomes)
    - Responsáveis não resolvidos (esperado: 0) — listar CNPJ + valor da coluna 4
    - Linhas com regime não mapeável (esperado: 0)

    Apply: para cada linha a criar (não pulada, responsável resolvido, regime mapeável), chamar:

    ```
    db.empresa.create({
      data: {
        nome,
        cnpj,                     // formato original da planilha "00.000.000/0000-00"
        regimeTributario,          // enum mapeado
        responsavelId,             // id do Usuario resolvido pelo nome
        ativo: true,
        contatos: null,
        particularidades,          // null ou "MEI"
        regimeHistorico: {
          create: {
            regimeTributario,
            dataInicio: new Date(),
          },
        },
      },
    })
    ```

    Apply output:
    - Total criadas, total puladas, total de responsáveis não resolvidos
    - Se `responsaveisNaoResolvidos.length > 0`: imprimir lista e `process.exit(1)` — não criar empresas com responsavelId inválido; o exit code sinaliza falha para o executor

    Verificação pós-apply (imprimir ao final do modo apply):
    - `await db.empresa.count()` → deve ser ≥ 197
    - `await db.empresa.groupBy({ by: ['responsavelId'], _count: true })` → imprimir distribuição por responsável
    - `await db.empresa.groupBy({ by: ['regimeTributario'], _count: true })` → imprimir distribuição por regime

    Após criar o script, rodar o dry-run:

    `node --env-file=.env scripts/importar-empresas.mjs`

    Imprimir a saída completa do dry-run no terminal para revisão humana.
